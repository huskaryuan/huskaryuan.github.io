  3. 数据倾斜优化
    1. map join 和 semi join
    2. 减少 join 的使用
      1. 宽表加工 union all 代替 join
      2. 某些维度和指标避免自关联的 sql 技巧（例如同时购买A商品和B商品的用户 n天复购率等等），
    3. join key 值分布不均匀（随机数打散，抽取）
    4. count distinct 优化
      1. 尽量不用count distinct
      2. 避免重复count distinct同一字段的情况，如有必要，可先count distinct一次计算原子指标，然后再和维度取值结合形成派生指标。
      3. groupby.skewdata = True
    5. 调参
  4. 数据量实在太大
    1. 从业务上看看能不能减少输入数据量
    2. 使用二级分区
    3. 适当的时候可以set exec.parallel = true 提高并发
